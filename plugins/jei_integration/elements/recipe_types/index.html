Recipe Types

On the Properties tab the basic settings for the JEI category can be set.

The JEI Texture is the texture that will be used as the background in the JEI category. The texture file has to be 256 x 256 pixel in size. The actual dimensions can be set using x, y, width and height fields.

The x and y set the texture position in the texture file and start from the top left and increase when going down and right. The width and height set the size of the part of the texture that will be used and start from the x and y position and add onto them.



The title and displayed icon will be the title and corresponding icon that will be shown in your JEI category.

The recipe blocks are all the blocks that support the recipe processing. For example if a mod adds a new furnace, that furnace will also be displayed in the vanilla furnace smelting category.

The Recipe Slots tab is the main tab to set what inputs and outputs the recipes can have and where they will be displayed in JEI.

Every slot has the IO option which determites, if the slot is either an input, output or a render. The input and output slots also require a type, which can either be an item, fluid, logic, number or text type, as well as a name, which has to be unique for each set slot.

Item and fluid slots also require an x and y coordinate, to set where they are located in the JEI category. Their coordinates follow the same principle as the JEI Texture.



Fluid slots have extra options to change their appearance in JEI. Their height can be set to appear above the normal slot height of 16 pixel.

The Use Full Tank option determites if the fluid is filled to the top of the slot, or has a size based on the size of the tank and the amount specified in a recipe.



Slots can also be set to optional, meaning they are not requires in a recipe.

Logic, number and text slots require a default value, that is used when they are not set in a recipe.



Render slots are special slots as they are not used by recipes, but act as additional textures inside JEI categories. Like other slots they need a unique name, but only require a texture to be selected. They can be customized and animated using the blocks in the Render GUI tab.

The Render GUI tab allows for extra rendered components, like text or textures, inside a JEI category.

While the basic blocks, like the normal logic, number and text blocks from the normal procedures are provided, the render editor also has some custom blocks.


The first one are the Category Rendering blocks which are the main blocks to add extra renders to the JEI category.

The Category Variables can be used to get the logic, number and text variables set in the recipe.

Lastly the Misc Blocks for some extra usage.

The Clickable Areas tab are an extra utility to connect GUIs to recipe types.

Clickable Areas are fields in GUIs that, when clicked, open the JEI category and show all recipes in that category

To create a clickable area select a gui first, than specify the x and y postion and the width and height size of the area.
